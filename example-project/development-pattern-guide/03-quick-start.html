Chapter 3

Quick Start Guide

Build and run a working layered application end to end. Explanations come later; this chapter is about seeing it work.

Step 1

Create the Project

A console application and its two package references.

dotnet new console -n LayeredApp
cd LayeredApp
dotnet add package Microsoft.EntityFrameworkCore.InMemory
dotnet add package Microsoft.Extensions.DependencyInjection

This produces a project with an entry point and the two libraries the pattern needs. The rest of the quick start adds one file per layer. Do not worry about why each file looks the way it does yet; the chapters that follow take each one apart.

Step 2

Add the Layers

A model, a database context, a repository, and two services.

Create the following files. Each is the smallest version of its role; later chapters expand on the reasoning.

// Entity.cs: the domain model
public class Entity
{
    public Guid Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public int Value { get; set; }
}
// DataContext.cs: the ORM context
using Microsoft.EntityFrameworkCore;

public class DataContext(DbContextOptions<DataContext> options) : DbContext(options)
{
    public DbSet<Entity> Entities => Set<Entity>();
}
// IRepository.cs and Repository.cs: the persistence boundary
public interface IRepository
{
    void AddRange(IEnumerable<Entity> entities);
    List<Entity> GetAll();
}

public class Repository(DataContext context) : IRepository
{
    public void AddRange(IEnumerable<Entity> entities)
    {
        context.Entities.AddRange(entities);
        context.SaveChanges();
    }

    public List<Entity> GetAll() => [.. context.Entities];
}
// IOrchestrationService.cs and OrchestrationService.cs: the use case
public interface IOrchestrationService
{
    void Run();
}

public class OrchestrationService(IRepository repository) : IOrchestrationService
{
    public void Run()
    {
        repository.AddRange(
        [
            new Entity { Id = Guid.NewGuid(), Name = "First", Value = 10 },
            new Entity { Id = Guid.NewGuid(), Name = "Second", Value = 20 },
        ]);

        foreach (Entity entity in repository.GetAll())
        {
            Console.WriteLine($"{entity.Name}: {entity.Value}");
        }
    }
}
Step 3

Wire and Run

The composition root that registers the layers and starts the program.

Replace the generated Program.cs with the composition root. It registers each interface against its implementation, builds the provider, creates the database, and resolves the top-level service to run.

// Program.cs: the composition root
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;

ServiceCollection services = new();

services.AddDbContext<DataContext>(options =>
    options.UseInMemoryDatabase("AppDb"));

services.AddScoped<IRepository, Repository>();
services.AddScoped<IOrchestrationService, OrchestrationService>();

using ServiceProvider provider = services.BuildServiceProvider();

DataContext context = provider.GetRequiredService<DataContext>();
context.Database.EnsureCreated();

IOrchestrationService service = provider.GetRequiredService<IOrchestrationService>();
service.Run();
dotnet run
# First: 10
# Second: 20

If you see the two lines printed, the pattern is working: the composition root built the object graph, injected the repository into the service, and the service drove the run without ever constructing a dependency itself. The remaining chapters explain each layer and why it is shaped this way.
