Chapter 3

Quick Start

Stand up a minimal working pipeline and run it — explanations come later.

Step 1

Create the project

A single console application will host every layer.

Create a new console project and move into it:

dotnet new console -n DataPipeline
cd DataPipeline
Step 2

Add the two packages

One for the container, one for in-memory persistence.

Add the dependency-injection container and the ORM's in-memory provider:

dotnet add package Microsoft.Extensions.DependencyInjection
dotnet add package Microsoft.EntityFrameworkCore.InMemory
Tip

The package versions should match your SDK's major version. If a build warns about a mismatch, install the version that lines up with your runtime.

Step 3

Provide a data source

A small JSON file gives the importer something to read.

Create Data/source.json with a couple of records, and make sure it is copied next to the binaries on build:

[
  { "id": "11111111-1111-1111-1111-111111111111", "name": "First",  "label": "Alpha", "amount": 100 },
  { "id": "22222222-2222-2222-2222-222222222222", "name": "Second", "label": "Beta",  "amount": 250 }
]
<!-- in the .csproj -->
<ItemGroup>
  <None Update="Data/source.json">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
Step 4

Wire the composition root

Register every layer, then ask the container for the orchestration service.

The entry point registers each interface against its implementation, builds the provider, and runs the pipeline. The classes referenced here are built one chapter at a time in the rest of the guide.

ServiceCollection services = new();

services.AddDbContext<PersistenceContext>(options =>
    options.UseInMemoryDatabase("PipelineDb"));

services.AddScoped<IRepository, Repository>();
services.AddScoped<ISourceImporter, SourceImporter>();
services.AddScoped<IOrchestrationService, OrchestrationService>();

using ServiceProvider provider = services.BuildServiceProvider();

IOrchestrationService service = provider.GetRequiredService<IOrchestrationService>();
service.ImportAndDisplay();
Step 5

Run it

One command builds, restores, and executes the pipeline.

dotnet run

You should see each imported record printed to the console. If you do, the full pipeline is working end to end. The chapters that follow rebuild each layer slowly so you understand exactly why it works.
