Chapter 4

High Level Architecture

The layers, the direction their dependencies point, and how one run flows through them.

The Big Picture

System Map

Click any node to expand it. Arrows show how each component depends on the next.

  • The Composition Root resolves and runs the Orchestration Service.
  • The Orchestration Service imports entities via the Import Service and persists them via the Persistence System.
  • The Import Service produces Domain Model entities; the Persistence System stores Domain Model entities.
  • Inside the Persistence System, the Repository saves through the Database Context.
Dependency Direction

Which Way the Arrows Point

Why every dependency flows toward an interface, never toward a concrete class.

The arrows in the map are dependencies, and their direction is the whole discipline of the pattern. The Orchestration Service depends on the IRepository and IImportService interfaces, not on the classes that implement them. The Repository depends on the Database Context. None of these objects construct their collaborators; each receives them through its constructor.

Because the dependencies target interfaces, the concrete graph is assembled in exactly one place: the Composition Root on the far left. It is the only component that names concrete classes. Every other component could be compiled against the interfaces alone. That is what makes a layer replaceable: swap the registration in the root, and nothing upstream changes.

One Run

Flow of a Single Execution

The sequence of calls from program start to printed output.

Reading the map left to right also reads the runtime sequence. The Composition Root builds the provider and resolves the Orchestration Service. The service asks the Import Service for entities, which reads the external source and deserializes it into Domain Models. The service hands those entities to the Persistence System, which writes them through the Database Context and can read them back. The service formats the retrieved entities into output. Control returns to the root, the provider is disposed, and the program ends.

Every step crosses exactly one interface boundary, and no step reaches around a layer to touch a layer two levels down. That restraint is what keeps each layer independently testable and independently replaceable.

Component

Composition Root

The only component that knows the concrete types. It registers each interface against its implementation in the DI container, builds the provider, creates the database, and resolves the Orchestration Service to start the run. Covered in Chapter 11.

Component

Orchestration Service

The use case. It receives the Import Service and the Repository through its constructor and calls them in order: import, persist, retrieve, output. It contains the control flow but no persistence or parsing detail. Covered in Chapter 10.

Component

Import Service

The boundary to the outside world. It locates the external source, reads it, deserializes it into Domain Models, and validates that the source exists. Replacing the source format means replacing this one class. Covered in Chapter 9.

System

Persistence System

Collapsed in the main map to keep it readable. Inside, a Repository exposes a narrow persistence interface and delegates to a Database Context, which is the ORM's view of the store. Callers see only the Repository interface; the Database Context never leaks upward.

Component

Domain Model

A plain object with fields and mapping attributes, and no behavior. It is the single shape that the Import Service produces, the Persistence System stores, and the Orchestration Service formats. Covered in Chapter 6.

Component

Repository

Exposes only the persistence operations the application needs (add, read, count) and hides the ORM behind them. Callers depend on its interface, so the backing store can change without touching them. Covered in Chapter 8.

Component

Database Context

The ORM context. It declares which entities are persisted and configures how each maps to a table. It is constructed by the DI container from options set in the Composition Root. Covered in Chapter 7.
