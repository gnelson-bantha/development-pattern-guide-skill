Chapter 3

Prerequisites

A short list of what to install and what to already be comfortable with before you start building the layers.

The Toolchain

SDK and Runtime

One install gets you the compiler, the runtime, and the command-line tooling in a single package.

You need the .NET SDK, version 10 or newer. The sample targets net10.0, and the SDK ships with everything you'll touch here: the C# compiler, the runtime that executes the program, and the dotnet command line for creating, building, and running projects. There's no separate runtime to chase down and no build server to stand up.

Confirm it's there before you write a line of code:

dotnet --version
# should print 10.x or higher

An editor with C# support makes the interfaces-everywhere style far more pleasant, because jumping from IRepository to its implementation becomes one keystroke instead of a folder hunt. Anything with the C# language tooling works; the pattern doesn't care which one you pick.

The Packages

Two NuGet Dependencies

The entire pattern leans on a persistence library and a container, and nothing more.

You'll add exactly two packages. The first is an Entity Framework Core provider, which is the object-relational mapper that turns your records into database rows. We use the in-memory provider here because it needs zero setup, but the point of the pattern is that you could swap it for SQLite or SQL Server later without disturbing the layers above it.

The second is the dependency injection container, which is the engine that reads your registrations and builds the object graph. Add both from the project directory:

dotnet add package Microsoft.EntityFrameworkCore.InMemory
dotnet add package Microsoft.Extensions.DependencyInjection

Why in-memory

The in-memory provider keeps the focus on structure instead of connection strings. Everything you learn about wiring the repository and context transfers directly to a real database; you change one registration line and the rest of the code never finds out.

The Background

What You Should Already Know

This is a structure lesson, not a first-day-of-C# lesson, so a few fundamentals are assumed.

The guide assumes you can read and write C# comfortably. Specifically, you'll be on solid ground if interfaces and classes are second nature, if you've seen constructor parameters used to pass dependencies in, and if generics like List<T> don't slow you down. We use a few modern C# conveniences without much ceremony: primary constructors (the parameters right on the class declaration), collection expressions like [.. items], and top-level statements in Program.cs.

You don't need prior Entity Framework experience. We introduce the context and the repository from scratch, and the persistence chapter explains the handful of EF concepts you actually touch. You also don't need to know dependency injection going in; the composition root chapter builds that idea up from the constructors you'll already have written.

That's the whole checklist. With the SDK installed and the two packages ready, the next chapter steps back to show how all the layers connect before we build them one by one.
