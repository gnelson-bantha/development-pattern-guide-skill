Pattern Tutorial

Chapter 2

Prerequisites

The runtime, packages, and background knowledge assumed by the rest of the guide.

Toolchain

Runtime and SDK

What you install to build and run the project.

The pattern is language-agnostic, but this guide implements it in C# on a current .NET SDK, because the platform ships a dependency injection container and an ORM that make the pattern concrete with very little ceremony. Install a recent .NET SDK and confirm it is on your path:

dotnet --version
# 10.0.x or any current SDK

The SDK includes the compiler, the package manager, and the dotnet command used to create, restore, build, and run the project. No IDE is required; a text editor and a terminal are enough.

Dependencies

Packages

The two libraries that supply the ORM and the DI container.

The pattern relies on two external libraries. The first is an ORM with an in-memory provider, which lets you store and query entities without standing up a real database. The second is a dependency injection container, which holds the registrations and constructs the object graph.

<ItemGroup>
  <PackageReference Include="Microsoft.EntityFrameworkCore.InMemory" Version="10.0.0" />
  <PackageReference Include="Microsoft.Extensions.DependencyInjection" Version="10.0.0" />
</ItemGroup>

The in-memory provider is chosen deliberately for a tutorial: it removes setup so the focus stays on the pattern. Because the repository hides the ORM behind an interface, swapping the in-memory provider for a real database provider later is a one-line change in the composition root, and no other layer is affected. That substitutability is the point of the pattern, and you get to use it the first time you outgrow the in-memory store.

Assumed Knowledge

Skills You Should Bring

What this guide does not stop to teach.

Interfaces and Classes

You should be comfortable defining an interface and implementing it in a class. The pattern uses an interface at every layer boundary.

Constructors

Dependencies arrive through constructor parameters. You should understand how an object receives values when it is created.

Generics and Collections

Lists, enumerables, and a generic method or two appear throughout. You do not need to write generics, only to read them.

JSON and Serialization

The import layer reads JSON. A basic understanding of mapping JSON fields to object properties is enough.

You do not need prior experience with this specific ORM or DI container. Each is introduced in the chapter where it first appears, with the reasoning behind every call rather than just the call itself.
