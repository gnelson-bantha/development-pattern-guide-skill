Chapter 2

Prerequisites

The tools, packages, and background knowledge you need before building the pipeline.

Toolchain

What to install

A modern .NET SDK is all the tooling required.

  • .NET SDK 10 or later — provides the compiler, the dotnet CLI, and the runtime that executes the pipeline.
  • A code editor — any editor with C# support works; one with language-server tooling makes navigating the layers easier.
  • A terminal — you will build and run the project from the command line with a single command.

Confirm the SDK is available before continuing:

dotnet --version
# Expect 10.x.x or newer
Packages

Libraries the pattern relies on

Two small packages supply the container and the persistence layer.

The pipeline depends on two well-known libraries, both restored automatically the first time you build:

  • A dependency-injection container — registers each interface against its implementation and resolves the graph at startup.
  • An object–relational mapper with an in-memory provider — backs the repository so you can run the whole pipeline with no external database to install.
No database required

The in-memory provider keeps the Quick Start friction-free. Swapping it for a real database is covered in Extending the System.

Assumed Skills

What you should already know

Comfort with a few core ideas will make the chapters land faster.

  • Reading and writing classes and interfaces in a typed language.
  • The basic idea of serialization — turning text such as JSON into objects and back.
  • A first acquaintance with dependency injection: the notion that an object receives its collaborators rather than creating them. The pattern chapters reinforce this from the ground up.
