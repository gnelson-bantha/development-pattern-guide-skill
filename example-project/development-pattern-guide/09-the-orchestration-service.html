Chapter 9

The Orchestration Service

Coordinate import, persist, and present in one place — depending only on abstractions.

Step 1

Expose a single entry method

The interface describes the whole job in one verb.

The orchestration service is the layer the application actually runs. Its interface is tiny — one method that runs the end-to-end flow. Keeping it small means the composition root has exactly one thing to resolve and call.

public interface IOrchestrationService
{
    void ImportAndDisplay();
}
Step 2

Receive collaborators through the constructor

The service asks for interfaces and never constructs them.

This is the heart of dependency injection. The service declares what it needs — an importer and a repository — as constructor parameters typed to their interfaces. It never calls new on either one. Whoever builds the service decides which concrete classes arrive.

public class OrchestrationService(
    ISourceImporter importer,
    IRepository repository) : IOrchestrationService
{
    // ImportAndDisplay follows in the next step.
}
Step 3

Drive the flow

Import, persist, read back, present — in that order.

The method reads as a description of the pipeline itself: ask the importer for records, hand them to the repository, read the stored records back, and present them. Every call goes through an interface, so the service has no knowledge of files, databases, or formats.

public void ImportAndDisplay()
{
    List<DataRecord> imported = importer.LoadFromSource();
    repository.AddRange(imported);

    List<DataRecord> stored = repository.GetAll();

    Console.WriteLine($"Loaded {repository.Count()} records from the store:");
    Console.WriteLine();

    foreach (DataRecord record in stored)
    {
        Console.WriteLine($"Id:     {record.Id}");
        Console.WriteLine($"Name:   {record.Name}");
        Console.WriteLine($"Label:  {record.Label}");
        Console.WriteLine($"Amount: {record.Amount:N0}");
        Console.WriteLine(new string('-', 40));
    }
}
Why this layer earns its place

The orchestration service is the only layer that knows the order of operations. Pulling that sequence out of the entry point keeps the composition root focused purely on wiring, and makes the flow easy to test with fake collaborators.

Takeaway

Logic that depends on nothing concrete

Every layer now exists; one step remains — connecting them.

The model, importer, repository, context, and service are all written and all depend only on abstractions. The final chapter assembles them in the composition root and presses go.
