Chapter 7

The Source Importer

Build the inbound layer that reads an external source and turns it into domain models.

Step 1

Define the interface first

The contract is what the rest of the pipeline depends on.

Every layer in this pattern begins as an interface. The importer's job is narrow: produce a list of domain models. Nothing in the contract mentions files, JSON, or any other source detail — that freedom is what lets you swap implementations later.

public interface ISourceImporter
{
    List<DataRecord> LoadFromSource();
}
Step 2

Locate the source reliably

Resolve the path relative to the running binaries, not the working directory.

The implementation reads a file that ships alongside the application. Build the path from the binary's base directory so the program finds it no matter where it is launched from.

public class SourceImporter : ISourceImporter
{
    private static readonly JsonSerializerOptions SerializerOptions = new()
    {
        PropertyNameCaseInsensitive = true
    };

    private readonly string _sourcePath =
        Path.Combine(AppContext.BaseDirectory, "Data", "source.json");

    // LoadFromSource follows in the next step.
}
Step 3

Read, validate, and deserialize

Fail loudly if the source is missing; return an empty list if it is empty.

Guard the one boundary condition that matters — a missing source — with a clear error. Deserialize the contents into domain models and return an empty list rather than null so callers never have to null-check.

public List<DataRecord> LoadFromSource()
{
    if (!File.Exists(_sourcePath))
    {
        throw new FileNotFoundException(
            $"Could not find the source data file at '{_sourcePath}'.",
            _sourcePath);
    }

    string json = File.ReadAllText(_sourcePath);
    List<DataRecord>? records =
        JsonSerializer.Deserialize<List<DataRecord>>(json, SerializerOptions);

    return records ?? [];
}
Boundary discipline

Validation belongs at the edges of the system. The importer is an edge, so it checks that the source exists. Inner layers can then trust the data they receive instead of re-checking it.

Takeaway

An adapter behind a contract

The outside world is now isolated to one swappable class.

The importer is the only place that knows where data comes from. Replace it with one that calls a network service or reads a different format, and the rest of the pipeline is untouched. Next you build the layer that stores these records.
